# US-11: Accessibility and SEO (Optional)

## User Story

**As a** user with assistive technology or a search engine  
**I want** the page to have sensible structure and semantics  
**So that** the content is accessible and indexable.

## Acceptance Criteria

- [ ] Heading hierarchy is logical (e.g., one H1, then H2, then H5 or H3 as appropriate; no skipped levels where avoidable).
- [ ] Links have descriptive text or accessible names (no "click here" only where avoidable).
- [ ] Images, if present in the main content, have appropriate alt text or decorative handling.
- [ ] Page has a meaningful `<title>` and meta description (or equivalent) for SEO.
- [ ] Focus order and keyboard navigation allow reaching main links and CTAs (optional, can be expanded in dedicated a11y tests).
