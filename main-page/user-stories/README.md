# Main Page — User Stories

User stories for the t23b.org homepage, formatted for US Agile (e.g., Scrum/SAFe).

## Format

Each story file includes:

- **Title** — Short name and ID (e.g., US-01).
- **User Story** — Standard format: *As a [role], I want [goal], So that [benefit].*
- **Acceptance Criteria** — Checklist of conditions that must be met for the story to be "Done."
- **Notes** — Scope and test context.

## File Naming

- `US-NN-<short-title>.md` — One file per story for easy import into Jira, Azure DevOps, or similar.
